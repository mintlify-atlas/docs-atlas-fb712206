The blog system supports rich content authoring with MDX, custom components, syntax highlighting, search integration, and RSS feeds.

## Blog Collection Schema

The blog collection is defined in `src/content.config.ts:49`:

```typescript
const blog = defineCollection({
  loader: glob({ pattern: "**/*.{md,mdx}", base: "./src/content/blog" }),
  schema: z.object({
    title: z.string(),
    date: z.coerce.date(),
    updatedDate: z.coerce.date().optional(),
    excerpt: excerptSchema,
    tags: tagsSchema,
  }),
});
```

## Reading Time Calculation

Reading time is automatically calculated for Japanese content using character count.

### Implementation

From `src/utils/reading-time.ts`:

```typescript
/**
 * Calculate reading time for content
 * @param content - The text content to analyze
 * @param wordsPerMinute - Reading speed (default: 400 for Japanese)
 * @returns Reading time in minutes
 */
export const getReadingTime = (
  content: string,
  wordsPerMinute = 400,
): number => {
  // For Japanese text, count characters instead of words
  // Remove HTML tags, code blocks, and whitespace
  const cleanContent = content
    .replace(/<[^>]*>/g, "") // Remove HTML tags
    .replace(/```[\s\S]*?```/g, "") // Remove code blocks
    .replace(/`[^`]*`/g, "") // Remove inline code
    .replace(/\s+/g, ""); // Remove whitespace

  // Calculate based on character count for Japanese
  const characters = cleanContent.length;
  const minutes = Math.ceil(characters / wordsPerMinute);

  return Math.max(1, minutes);
};
```

<Note>
  The function uses 400 characters per minute as the default reading speed, optimized for Japanese text.
</Note>

### Usage Example

```typescript
import { getReadingTime, formatReadingTime } from '@/utils/reading-time';

const content = await entry.render();
const readingTime = getReadingTime(content.body);
const formatted = formatReadingTime(readingTime); // "約3分で読了"
```

## RSS Feed Generation

The blog generates an RSS feed at `/rss.xml` for feed readers.

### RSS Endpoint

From `src/pages/rss.xml.ts`:

```typescript
import { getCollection } from "astro:content";
import rss from "@astrojs/rss";
import type { APIContext } from "astro";

export async function GET(context: APIContext) {
  const blog = await getCollection("blog");
  const sortedPosts = blog.sort(
    (a, b) => b.data.date.getTime() - a.data.date.getTime(),
  );

  return rss({
    title: "ta93abe | Blog",
    description: "技術ブログ。日々の学びや開発の記録を共有しています。",
    site: context.site?.toString() ?? "http://localhost:4321",
    items: sortedPosts.map((post) => ({
      title: post.data.title,
      pubDate: post.data.date,
      description: post.data.excerpt,
      link: `/blog/${post.id}/`,
    })),
    customData: "<language>ja</language>",
  });
}
```

## Creating a New Blog Post

<Steps>
  <Step title="Create MDX file">
    Create a new file in `src/content/blog/` with `.mdx` extension:
    
    ```bash
    src/content/blog/my-new-post.mdx
    ```
  </Step>
  
  <Step title="Add frontmatter">
    Include required fields at the top of the file:
    
    ```mdx
    ---
    title: "My New Blog Post"
    date: 2026-03-04
    excerpt: "A brief description of what this post covers"
    tags: ["TypeScript", "Astro"]
    ---
    ```
  </Step>
  
  <Step title="Import components (optional)">
    Import any components you want to use:
    
    ```mdx
    import Callout from "../../components/blog/Callout.astro";
    import LinkCard from "../../components/blog/LinkCard.astro";
    ```
  </Step>
  
  <Step title="Write content">
    Write your post using Markdown, MDX components, and code blocks.
  </Step>
  
  <Step title="Build and preview">
    The post will automatically appear on the blog index page, sorted by date.
    
    ```bash
    npm run dev
    ```
  </Step>
</Steps>
