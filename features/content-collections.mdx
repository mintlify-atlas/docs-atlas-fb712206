## Common Schema Parts

Shared schema definitions used across collections:

```typescript
// Tags schema - consistent across collections
const tagsSchema = z.array(z.string()).optional();

// Excerpt schema - short description
const excerptSchema = z.string();
```

## Local Content Collections

These collections use the `glob` loader to read files from the filesystem.

### Blog Collection

<ParamField path="loader" type="glob">
  Pattern: `**/*.{md,mdx}` in `src/content/blog`
</ParamField>

**Schema:**

```typescript
const blog = defineCollection({
  loader: glob({ pattern: "**/*.{md,mdx}", base: "./src/content/blog" }),
  schema: z.object({
    title: z.string(),
    date: z.coerce.date(),
    updatedDate: z.coerce.date().optional(),
    excerpt: excerptSchema,
    tags: tagsSchema,
  }),
});
```

<Tabs>
  <Tab title="Field Types">
    <ParamField path="title" type="string" required>
      Blog post title
    </ParamField>
    <ParamField path="date" type="date" required>
      Publication date (coerced from string)
    </ParamField>
    <ParamField path="updatedDate" type="date">
      Last updated date (optional)
    </ParamField>
    <ParamField path="excerpt" type="string" required>
      Short description for previews and SEO
    </ParamField>
    <ParamField path="tags" type="string[]">
      Optional array of tags for categorization
    </ParamField>
  </Tab>
  <Tab title="Example Frontmatter">
    ```mdx
    ---
    title: "MDXブログ機能のデモ"
    date: 2026-01-11
    excerpt: "このブログで使える機能を紹介します。コードハイライト、コールアウト、テーブルなど。"
    tags: ["MDX", "Astro", "ブログ"]
    ---

    import Callout from "../../components/blog/Callout.astro";
    import LinkCard from "../../components/blog/LinkCard.astro";

    このブログではMDXを使って、リッチなコンテンツを書くことができます。
    ```

  </Tab>
</Tabs>

### Works Collection

<ParamField path="loader" type="glob">
  Pattern: `**/*.{md,mdx}` in `src/content/works`
</ParamField>

**Schema:**

```typescript
const works = defineCollection({
  loader: glob({ pattern: "**/*.{md,mdx}", base: "./src/content/works" }),
  schema: ({ image }) =>
    z.object({
      title: z.string(),
      coverImage: image(),
      tags: z.array(z.string()),

```

### Books Collection

<ParamField path="loader" type="glob">
  Pattern: `**/*.md` in `src/content/books`
</ParamField>

**Schema:**

```typescript
const books = defineCollection({
  loader: glob({ pattern: "**/*.md", base: "./src/content/books" }),
  schema: ({ image }) =>
    z.object({
      title: z.string(),
      author: z.string(),
      coverImage: image(),
      status: z.enum(["read", "reading", "stacked"]),
      finishedDate: z.coerce.date().optional(),
      rating: z.number().min(1).max(5).optional(),
      category: z.string().optional(),
      tags: z.array(z.string()),
      excerpt: excerptSchema,
    }),
});
```

      Reading status: `read`, `reading`, or `stacked`

    <ParamField path="finishedDate" type="date">

### Talks Collection

<ParamField path="loader" type="glob">
  Pattern: `**/*.json` in `src/content/talks`
</ParamField>

**Schema:**

```typescript
const talks = defineCollection({
  loader: glob({ pattern: "**/*.json", base: "./src/content/talks" }),
  schema: z.object({
    items: z.array(
      z.object({
        id: z.string(),
        url: z.string().url().optional(),
        date: z.coerce.date(),
        title: z.string(),
      }),
    ),
  }),
});
```

<Tabs>
  <Tab title="Field Types">
    <ParamField path="items" type="array" required>
      Array of talk objects
    </ParamField>
    <ParamField path="items[].id" type="string" required>
      Unique identifier for the talk
    </ParamField>
    <ParamField path="items[].title" type="string" required>
      Talk title
    </ParamField>
    <ParamField path="items[].date" type="date" required>
      Presentation date
    </ParamField>
    <ParamField path="items[].url" type="string">
      Link to slides or recording (optional)
    </ParamField>
  </Tab>
  <Tab title="Example JSON">
    ```json
    {
      "items": [
        {
          "id": "talk-1",
    ```

<Note>
  The talks collection uses a single JSON file with an array of items, rather than individual files per talk.
</Note>

## External Content Collections

These collections use the custom RSS loader to fetch content from external sources.

### External Article Schema

All external collections share a common schema factory:

```typescript
const externalArticleSchema = (source: RSSSource) =>
  z.object({
    title: z.string(),
    url: z.string().url(),
    publishedAt: z.date(),
    excerpt: z.string(),
    source: z.literal(source),
  });
```

### Zenn Collection

```typescript
const zenn = defineCollection({
  loader: createRSSLoader("https://zenn.dev/ta93abe/feed", "Zenn", "zenn"),
  schema: externalArticleSchema("Zenn"),
});
```

### Note Collection

```typescript
const note = defineCollection({
  loader: createRSSLoader("https://note.com/ta93abe/rss", "Note", "note"),
  schema: externalArticleSchema("Note"),
});
```

### Podcast Collection

```typescript
const podcast = defineCollection({
  loader: createRSSLoader(
    "https://anchor.fm/s/4dd661e8/podcast/rss",
    "Podcast",
    "podcast",
  ),
  schema: externalArticleSchema("Podcast"),
});
```

<ParamField path="title" type="string">
  Article or episode title from RSS feed
</ParamField>
<ParamField path="url" type="string">
  Link to the external content
</ParamField>
<ParamField path="publishedAt" type="date">
  Publication date from RSS feed
</ParamField>
<ParamField path="excerpt" type="string">
  Content snippet or description
</ParamField>
<ParamField path="source" type="literal">
  Source identifier: `"Zenn"`, `"Note"`, or `"Podcast"`
</ParamField>
