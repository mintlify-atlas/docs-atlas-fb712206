The portfolio automatically fetches and displays content from external platforms using RSS feeds, powered by a custom RSS loader.

## RSS Loader Overview

The custom RSS loader (`src/utils/rss-loader.ts`) enables Astro Content Collections to fetch and parse RSS feeds as content sources.

### Architecture

```typescript
export type RSSSource = "Zenn" | "Note" | "Podcast";

interface RSSItem {
  id: string;
  title: string;
  url: string;
  publishedAt: Date;
  excerpt: string;
  source: RSSSource;
}
```

## RSS Loader Implementation

From `src/utils/rss-loader.ts`:

```typescript
import Parser from "rss-parser";

/**
 * Create an RSS feed loader for Astro content collections
 * @param url - The RSS feed URL
 * @param source - The source name (Zenn, Note, Podcast)
 * @param idPrefix - The prefix for generated IDs (e.g., "zenn", "note", "podcast")
 */
export const createRSSLoader = (
  url: string,
  source: RSSSource,
  idPrefix: string,
) => {
  return async (): Promise<RSSItem[]> => {
    try {
      const parser = new Parser();
      const feed = await parser.parseURL(url);

      if (!feed.items) {
        return [];
      }

      return feed.items
        .filter(
          (
            item,
          ): item is typeof item & {
            title: string;
            link: string;
            pubDate: string;
          } =>
            typeof item.title === "string" &&
            typeof item.link === "string" &&
            typeof item.pubDate === "string",
        )
        .map((item, index) => ({
          id: item.guid ?? `${idPrefix}-${index}`,
          title: item.title,
          url: item.link,
          publishedAt: new Date(item.pubDate),
          excerpt: item.contentSnippet ?? item.content ?? "",
          source,
        }));
    } catch (error) {
      console.error(`Error fetching ${source} RSS feed:`, error);
      return [];
    }
  };
};
```

## External Article Schema

All external content collections share a common schema factory:

```typescript
const externalArticleSchema = (source: RSSSource) =>
  z.object({
    title: z.string(),
    url: z.string().url(),
    publishedAt: z.date(),
    excerpt: z.string(),
    source: z.literal(source),
  });
```

<ParamField path="title" type="string" required>
  Article or episode title from the RSS feed
</ParamField>

<ParamField path="url" type="string" required>
  Full URL to the external content (validated as proper URL)
</ParamField>

<ParamField path="publishedAt" type="date" required>
  Publication date parsed from RSS pubDate field
</ParamField>

<ParamField path="excerpt" type="string" required>
  Content snippet or description from the feed
</ParamField>

<ParamField path="source" type="literal" required>
  Source identifier: `"Zenn"`, `"Note"`, or `"Podcast"`
</ParamField>

## Zenn Integration

[Zenn](https://zenn.dev/) is a Japanese platform for technical articles and books.

### Collection Definition

From `src/content.config.ts:86`:

```typescript
const zenn = defineCollection({
```

### Feed URL Structure

```
https://zenn.dev/{username}/feed
```

Replace `{username}` with your Zenn username.

### Usage Example

```typescript
import { getCollection } from 'astro:content';
```


## Fetching Content

External content is fetched at build time by Astro's content layer.

### Build-Time Fetching

```bash
npm run build
```

During build:
1. RSS loader fetches feeds from external URLs
2. Content is parsed and validated against schema
3. Items are stored as content collection entries
4. Pages are generated with the fetched content

### Development Mode

In dev mode, RSS feeds are fetched on server start:

```bash
npm run dev
```

<Note>
  Content is cached during the dev server session. Restart the server to fetch fresh content.
</Note>

## Combining External and Local Content

You can combine external content with local blog posts:

```typescript
import { getCollection } from 'astro:content';

// Get all content sources
const blog = await getCollection('blog');
const zenn = await getCollection('zenn');
const note = await getCollection('note');

// Combine and sort chronologically
const allArticles = [
  ...blog.map(post => ({
    title: post.data.title,
    date: post.data.date,
    url: `/blog/${post.id}/`,
    source: 'Blog',
    excerpt: post.data.excerpt,
  })),
  ...zenn.map(article => ({
    title: article.data.title,
    date: article.data.publishedAt,
    url: article.data.url,
    source: article.data.source,
    excerpt: article.data.excerpt,
  })),
  ...note.map(article => ({
    title: article.data.title,
    date: article.data.publishedAt,
    url: article.data.url,
    source: article.data.source,
    excerpt: article.data.excerpt,
  })),
].sort((a, b) => b.date.getTime() - a.date.getTime());
```

## Error Handling

The RSS loader includes robust error handling:

### Feed Fetch Failures

```typescript
try {
  const parser = new Parser();
  const feed = await parser.parseURL(url);
  // ...
} catch (error) {
  console.error(`Error fetching ${source} RSS feed:`, error);
  return []; // Return empty array on error
}
```

### Missing Fields

Type guards ensure required fields exist:

```typescript
.filter(
  (item): item is typeof item & {
    title: string;
    link: string;
    pubDate: string;
  } =>
    typeof item.title === "string" &&
    typeof item.link === "string" &&
    typeof item.pubDate === "string",
)
```

Items missing required fields are automatically filtered out.

## Adding New External Sources

To add a new external content source:

<Steps>
  <Step title="Update RSSSource type">
    Add your source to the type definition:
    
    ```typescript
    export type RSSSource = "Zenn" | "Note" | "Podcast" | "Medium";
    ```
  </Step>
  
  <Step title="Define collection">
    Add a new collection in `src/content.config.ts`:
    
    ```typescript
    const medium = defineCollection({
      loader: createRSSLoader(
        "https://medium.com/feed/@username",
        "Medium",
        "medium"
      ),
      schema: externalArticleSchema("Medium"),
    });
    ```
  </Step>
  
  <Step title="Export collection">
    Add to the collections export:
    
    ```typescript
    export const collections = {
      works,
      books,
      blog,
      talks,
      zenn,
      note,
      podcast,
      medium, // New collection
    };
    ```
  </Step>
  
  <Step title="Use in pages">
    Query and display the new content:
    
    ```typescript
    import { getCollection } from 'astro:content';
    const mediumPosts = await getCollection('medium');
    ```
  </Step>
</Steps>

## Best Practices

<Tabs>
  <Tab title="Feed URLs">
    - Use HTTPS URLs when available
    - Verify feed URL works before adding
    - Check feed update frequency
    - Consider rate limiting for frequently updated feeds
  </Tab>
  
  <Tab title="Error Handling">
    - Always return empty array on errors
    - Log errors for debugging
    - Don't crash the build on feed failures
    - Consider adding retry logic for transient failures
  </Tab>
  
  <Tab title="Performance">
    - RSS feeds are fetched at build time
    - No runtime performance impact
    - Consider caching for local development
    - Rebuild to fetch fresh content
  </Tab>
  
  <Tab title="Content">
    - Validate excerpt length and format
    - Handle missing content gracefully
    - Preserve external URLs correctly
    - Add source badges for visual distinction
  </Tab>
</Tabs>
