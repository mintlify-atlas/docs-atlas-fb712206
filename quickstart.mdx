Get ta93abe portfolio site up and running in just a few minutes. This guide will walk you through cloning the repository, installing dependencies, and starting the development server.

## Prerequisites

Before you begin, make sure you have the following installed:

- **Node.js** 18 or higher (tested with v24.13.1)
- **pnpm** package manager (recommended)

If you don't have pnpm installed, you can install it globally:

```bash
npm install -g pnpm
```

## Quick Setup

<Steps>
  <Step title="Clone the repository">
    Clone the ta93abe portfolio repository to your local machine:

    ```bash
    git clone <repository-url>
    cd me
    ```
  </Step>

  <Step title="Install dependencies">
    Install all required dependencies using pnpm:

    ```bash
    pnpm install
    ```

    This will install Astro, MDX, Tailwind CSS, and all other dependencies defined in package.json.
  </Step>

  <Step title="Start the development server">
    Launch the development server:

    ```bash
    pnpm dev
    ```

    Your site will be available at `http://localhost:4321`
  </Step>

  <Step title="Open in your browser">
    Navigate to `http://localhost:4321` to see your portfolio site running locally.
  </Step>
</Steps>

<Note>
  The development server supports hot module replacement (HMR), so changes you make to your code will be reflected immediately in the browser.
</Note>

## Available Commands

All commands are run from the root of the project:

<CodeGroup>
```bash Development
pnpm dev
```

```bash Build
pnpm build
```

```bash Preview
pnpm preview
```

```bash Format Code
pnpm format
```

```bash Lint
pnpm lint
```
</CodeGroup>

| Command | Description |
|---------|-------------|
| `pnpm dev` | Starts local dev server at `localhost:4321` |
| `pnpm build` | Build your production site to `./dist/` |
| `pnpm preview` | Preview your build locally, before deploying |
| `pnpm format` | Format code with Biome |
| `pnpm lint` | Lint code with Biome |
| `pnpm test` | Run Vitest unit tests |
| `pnpm test:e2e` | Run Playwright E2E tests |
| `pnpm deploy` | Build and deploy to Cloudflare |

## Create Your First Blog Post

Blog posts are stored in `src/content/blog/`. Let's create a new post:

<Steps>
  <Step title="Create a new markdown file">
    Create a new file in `src/content/blog/`, for example `my-first-post.md`:

    ```bash
    touch src/content/blog/my-first-post.md
    ```
  </Step>

  <Step title="Add frontmatter and content">
    Open the file and add the following content:

    ```markdown
    ---
    title: "最初のブログ投稿"
    date: 2024-01-01
    excerpt: "ブログを始めました。"
    ---

    # 最初のブログ投稿

    ブログを始めました。今後、技術的な内容や日々の学びを共有していきます。

    ## 今後の予定

    - Astroの使い方
    - TypeScriptのベストプラクティス
    - パフォーマンス最適化
    ```

    <Note>
      The frontmatter includes `title`, `date`, and `excerpt`. These fields are used to display post metadata throughout the site.
    </Note>
  </Step>

  <Step title="View your post">
    Navigate to `http://localhost:4321/blog` to see your new post in the blog listing.
  </Step>
</Steps>
