The portfolio integrates **Ethereum Name Service (ENS)** to resolve blockchain identities. The site uses `ta93abe.eth` as defined in the site configuration, allowing Web3 identity integration.

## What is ENS?

ENS (Ethereum Name Service) is a decentralized naming system built on Ethereum. It allows:

- Human-readable names instead of long addresses
- Associated metadata (avatar, social links, etc.)
- Decentralized profile management

## Implementation

The ENS utility (`src/utils/ens.ts`) uses **Viem**, a TypeScript library for Ethereum interactions.

### Setting Up the Client

A public Ethereum client is created to interact with the mainnet:

```typescript
import { createPublicClient, http } from "viem";
import { mainnet } from "viem/chains";

function getClient() {
  return createPublicClient({
    chain: mainnet,
    transport: http("https://cloudflare-eth.com"),
  });
}
```

This uses **Cloudflare's public Ethereum RPC endpoint**, which is free and doesn't require an API key.

### Profile Structure

The ENS profile includes various metadata fields:

```typescript
export interface EnsProfile {
  ensName: string;        // The ENS name (e.g., "ta93abe.eth")
  address: string;        // Ethereum address
  avatar: string | null;  // Avatar image URL
  description: string | null;  // Bio/description
  twitter: string | null;      // Twitter handle
  github: string | null;       // GitHub username
  url: string | null;          // Website URL
}
```

### Fetching ENS Profile

The main function resolves all ENS data in parallel (`src/utils/ens.ts:21-50`):

```typescript
export async function getEnsProfile(
  ensName: string,
): Promise<EnsProfile | null> {
  try {
    const client = getClient();
    
    // First, resolve the ENS name to an address
    const address = await client.getEnsAddress({ name: ensName });
    if (!address) return null;

    // Fetch all text records in parallel
    const [avatar, description, twitter, github, url] = await Promise.all([
      client.getEnsAvatar({ name: ensName }),
      client.getEnsText({ name: ensName, key: "description" }),
      client.getEnsText({ name: ensName, key: "com.twitter" }),
      client.getEnsText({ name: ensName, key: "com.github" }),
      client.getEnsText({ name: ensName, key: "url" }),
    ]);

    return {
      ensName,
      address,
      avatar,
      description,
      twitter,
      github,
      url,
    };
  } catch (error) {
    console.error(`[ENS] Failed to fetch profile for ${ensName}:`, error);
    return null;
  }
}
```

## ENS Text Records

ENS supports various standardized text record keys:

| Key | Description | Example |
|-----|-------------|----------|
| `description` | Bio or profile description | "Software Engineer" |
| `com.twitter` | Twitter username | "ta93abe" |
| `com.github` | GitHub username | "ta93abe" |
| `url` | Website URL | "https://ta93abe.com" |
| `email` | Email address | "hello@example.com" |
| `avatar` | Avatar image (special handling) | Resolved via `getEnsAvatar()` |

### Custom Text Records

You can fetch any custom text record:

```typescript
const customField = await client.getEnsText({
  name: "ta93abe.eth",
  key: "com.discord"
});
```

## Error Handling

The function gracefully handles errors and returns `null` if:

- The ENS name doesn't exist
- The network is unavailable
- The RPC endpoint fails

```typescript
try {
  const profile = await getEnsProfile("nonexistent.eth");
  if (!profile) {
    console.log("ENS name not found");
  }
} catch (error) {
  console.error("Failed to fetch ENS profile:", error);
}
```

## Performance Optimization

### Parallel Fetching

All ENS text records are fetched in parallel using `Promise.all()`, which reduces the total fetch time significantly:

```typescript
const [avatar, description, twitter, github, url] = await Promise.all([
  client.getEnsAvatar({ name: ensName }),
  client.getEnsText({ name: ensName, key: "description" }),
  client.getEnsText({ name: ensName, key: "com.twitter" }),
  client.getEnsText({ name: ensName, key: "com.github" }),
  client.getEnsText({ name: ensName, key: "url" }),
]);
```

### Caching Recommendations

For production use, consider caching ENS data:

```typescript
let cachedProfile: EnsProfile | null = null;
let cacheTime = 0;
const CACHE_DURATION = 1000 * 60 * 60; // 1 hour

export async function getCachedEnsProfile(
  ensName: string
): Promise<EnsProfile | null> {
  const now = Date.now();
  
  if (cachedProfile && (now - cacheTime) < CACHE_DURATION) {
    return cachedProfile;
  }
  
  cachedProfile = await getEnsProfile(ensName);
  cacheTime = now;
  
  return cachedProfile;
}
```
