## Meta Tags

The site includes comprehensive meta tags configured in `src/layouts/Layout.astro`.

### Primary Meta Tags

```astro
<title>{title}</title>
<meta name="title" content={title} />
<meta name="description" content={description} />
<meta name="author" content={SITE.author} />
<link rel="canonical" href={canonicalURL} />
<meta name="generator" content={Astro.generator} />
<meta name="color-scheme" content="light" />
```

### Canonical URLs

Every page includes a canonical URL to prevent duplicate content issues:

```astro
const canonicalURL = new URL(Astro.url.pathname, Astro.site);
```

```html
<link rel="canonical" href={canonicalURL} />
```

### Setting Page Meta Tags

Customize meta tags for each page by passing props to the Layout:

```astro
---
import Layout from "../layouts/Layout.astro";
---

<Layout
  title="Your Page Title"
  description="Your page description for SEO"
>
  <!-- Page content -->
</Layout>
```

## Structured Data (JSON-LD)

The site uses JSON-LD structured data for better search engine understanding.

### WebSite Schema

Every page includes WebSite schema from `src/utils/schema.ts`:

```typescript
export const generateWebSiteSchema = (
  siteUrl: string,
  options: WebSiteSchemaOptions = {},
): WebSiteSchema => {
  return {
    "@context": "https://schema.org",
    "@type": "WebSite",
    name: SITE.name,
    url: siteUrl,
    description: SITE.description,
    author: {
      "@type": "Person",
      name: SITE.author,
      url: siteUrl,
    },
    inLanguage: SITE.lang,
    potentialAction: {
      "@type": "SearchAction",
      target: {
        "@type": "EntryPoint",
        urlTemplate: `${siteUrl}tools?q={search_term_string}`,
      },
      "query-input": "required name=search_term_string",
    },
  };
};
```

### BreadcrumbList Schema

The schema utilities also support breadcrumb structured data:

```typescript
import { generateBreadcrumbSchema } from "../utils/schema";

const breadcrumbSchema = generateBreadcrumbSchema(
  [
    { label: "Home", href: "/" },
    { label: "Blog", href: "/blog" },
    { label: "Current Post" },
  ],
  Astro.site?.href || SITE.url
);
```

### Schema Safety

Schemas are safely stringified to prevent script injection:

```typescript
export const stringifySchema = (schema: object): string => {
  return JSON.stringify(schema).replace(/</g, "\\u003c");
};
```

## Sitemap Generation

The site automatically generates a sitemap at build time.

### Configuration

Sitemap is configured in `astro.config.mjs`:

```javascript
import sitemap from "@astrojs/sitemap";

export default defineConfig({
  site: "https://ta93abe.com",
  integrations: [sitemap()],
});
```

### Accessing the Sitemap

After building, the sitemap is available at:


## RSS Feed

The site includes an RSS feed for blog content at `src/pages/rss.xml.ts`.

### RSS Implementation

```typescript
import { getCollection } from "astro:content";
import rss from "@astrojs/rss";
import type { APIContext } from "astro";

export async function GET(context: APIContext) {
  const blog = await getCollection("blog");
  const sortedPosts = blog.sort(
    (a, b) => b.data.date.getTime() - a.data.date.getTime(),
  );

  return rss({
    title: "ta93abe | Blog",
    description: "技術ブログ。日々の学びや開発の記録を共有しています。",
    site: context.site?.toString() ?? "http://localhost:4321",
    items: sortedPosts.map((post) => ({
      title: post.data.title,
      pubDate: post.data.date,
      description: post.data.excerpt,
      link: `/blog/${post.id}/`,
    })),
    customData: "<language>ja</language>",
  });
}
```

### RSS Feed Location

```
https://your-site.com/rss.xml
```

## Performance Optimization

Performance is a key SEO ranking factor. The site includes several optimizations.

### HTML Compression

```javascript
export default defineConfig({
  compressHTML: true,
});
```

### CSS Optimization

```javascript
build: {
  inlineStylesheets: "auto", // Inline critical CSS
}

vite: {
  build: {
    cssCodeSplit: true, // Split CSS for better caching
  },
```

Improve perceived performance with hover-based prefetching:

```javascript
prefetch: {
  prefetchAll: false,
  defaultStrategy: "hover",
}
```
