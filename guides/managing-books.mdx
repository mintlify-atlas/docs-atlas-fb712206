## Overview

The bookshelf feature lets you maintain a reading list with statuses, ratings, and reviews. Each book is a `.md` file in the `src/content/books/` directory.

## Quick Start

1. Create a new `.md` file in `src/content/books/`
2. Add frontmatter with book details
3. Add a cover image to `public/`
4. Write your review or notes
5. The book appears on your bookshelf

## File Naming

Use the book title or a short identifier:

```
src/content/books/atomic-habits.md
src/content/books/clean-code.md  
src/content/books/designing-data-intensive.md
```

## Frontmatter Schema

The books schema is defined in `src/content.config.ts:32-46`. Most fields are required.

### Required Fields

<ParamField path="title" type="string" required>
  The book title
</ParamField>

<ParamField path="author" type="string" required>
  The book author (or authors)
</ParamField>

<ParamField path="coverImage" type="image" required>
  Path to the book cover image
  
  Example: `"../../../public/book-cover.jpg"`
</ParamField>

<ParamField path="status" type="enum" required>
  Current reading status. Must be one of:
  
  - `"read"` - Finished reading
  - `"reading"` - Currently reading
  - `"stacked"` - In your reading queue/to-read list
</ParamField>

<ParamField path="tags" type="string[]" required>
  Book topics or genres. Unlike blog posts, tags are required for books.
  
  Example: `["Programming", "Software Engineering", "Best Practices"]`
</ParamField>

<ParamField path="excerpt" type="string" required>
  A brief description or your key takeaway from the book (1-2 sentences)
</ParamField>

### Optional Fields

<ParamField path="finishedDate" type="date">
  Date you finished reading the book (YYYY-MM-DD format). Typically used with `status: "read"`
</ParamField>

<ParamField path="rating" type="number">
  Your rating from 1 to 5. Must be an integer between 1 and 5.
</ParamField>

<ParamField path="category" type="string">
  The book category or genre
  
  Example: `"Technical"`, `"Business"`, `"Fiction"`
</ParamField>

## Status Management

The `status` field tracks your reading progress:

### Read

Books you've completed:

```yaml
status: "read"
finishedDate: 2024-06-30
rating: 5
```

**Best practices:**
- Always include `finishedDate` for completed books
- Add a `rating` to help others discover great reads
- Write a review or notes in the content

### Reading

Books you're currently working through:

```yaml
status: "reading"
```

**Tips:**
- Update to `"read"` when finished
- Add notes or highlights as you go
- No need for `finishedDate` or `rating` yet

### Stacked

Books in your reading queue:

```yaml
status: "stacked"
```

**Use case:**
- Books you plan to read
- Recommendations you've collected
- Your "to-read" list

## Cover Images

### Adding Book Covers

1. **Save the image** to the `public/` directory:
   ```
   public/
     ├── atomic-habits.jpg
     ├── clean-code.png
     └── designing-data-intensive.jpg
   ```

2. **Reference in frontmatter**:
   ```yaml
   coverImage: "../../../public/atomic-habits.jpg"
   ```

### Image Guidelines

- **Format**: JPG or PNG
- **Dimensions**: Typical book cover ratio (e.g., 400×600px)
- **File size**: Keep under 200KB
- **Quality**: Use official book cover art when possible

### Finding Book Covers

Sources for book cover images:

- Google Books API
- Open Library
- Amazon product images
- Publisher websites
- Your own photos of physical books

## Rating System

Ratings must be integers from 1 to 5:

```yaml
rating: 5  # Excellent - highly recommend
rating: 4  # Very good - recommend
rating: 3  # Good - worth reading
rating: 2  # Okay - has some value
rating: 1  # Not recommended
```

**Tips:**

- Be consistent with your rating scale
- Ratings are optional but helpful
- Only rate books you've finished (`status: "read"`)
- Explain your rating in the review

## Categories and Tags

### Category

Broad classification of the book:

```yaml
category: "Technical"      # Programming, engineering
category: "Business"       # Leadership, management
category: "Fiction"        # Novels, stories
category: "Self-Help"      # Personal development
category: "Science"        # Popular science, research
```

Use one category per book.

### Tags

Specific topics covered in the book:

```yaml
# Technical book
tags: ["JavaScript", "Web Development", "Best Practices"]

# Business book  
tags: ["Leadership", "Team Management", "Communication"]

# Fiction
tags: ["Science Fiction", "Space Opera", "Adventure"]
```

**Best practices:**

- Use 3-5 tags per book
- Be specific but reusable
- Use title case
- Keep consistent across books

## Writing Book Reviews

The content section (after frontmatter) is for your review and notes.

### Suggested Structure

```markdown
# [Book Title]

[Opening paragraph - overall impression]

## Key Takeaways

- Main point 1
- Main point 2  
- Main point 3

## What I Liked

- Positive aspect 1
- Positive aspect 2

## What Could Be Better

- Critique 1
- Critique 2

## Who Should Read This

[Target audience and recommendations]

## Favorite Quotes

> "Memorable quote from the book"

## My Thoughts

[Personal reflection and application]
```

### Review Tips

<Callout type="tip" title="Be Specific">
  Instead of "good book," explain what made it valuable: specific insights, actionable advice, or memorable examples.
</Callout>

<Callout type="info">
  Your reviews help future readers decide if a book is right for them. Focus on content, clarity, and practical value.
</Callout>

## Complete Workflow

### Adding a New Book

1. **Create the file**
   ```bash
   touch src/content/books/my-new-book.md
   ```

2. **Add cover image**
   ```bash
   cp book-cover.jpg public/
   ```

3. **Write frontmatter**
   ```yaml
   ---
   title: "Book Title"
   author: "Author Name"
   coverImage: "../../../public/book-cover.jpg"
   status: "reading"
   category: "Technical"
   tags: ["Tag1", "Tag2", "Tag3"]
   excerpt: "Brief description of the book."
   ---
   ```

4. **Add review** (optional at first)
   ```markdown
   # Book Title
   
   Your notes and impressions...
   ```

5. **Preview**
   ```bash
   npm run dev
   ```

### Updating Status

When you finish a book:

1. Change status to `"read"`
2. Add `finishedDate`
3. Add `rating`
4. Write or complete your review

```yaml
# Before
status: "reading"

# After
status: "read"  
finishedDate: 2024-03-15
rating: 4
```

## Organizing Your Bookshelf

### By Status

Filter books by reading status to show:

- Currently reading
- Completed books
- Reading queue

### By Category

Group books by category:

- Technical books
- Business books
- Fiction
- Self-help

### By Rating

Highlight your top-rated books:

- 5-star recommendations
- 4+ star books

### By Date

Sort by `finishedDate` to show:

- Recently completed
- Reading timeline
- Yearly reading goals

## Tips and Best Practices

<Callout type="tip" title="Update Regularly">
  Add books as you start reading them. Update status and add reviews as you finish.
</Callout>

<Callout type="info">
  Use the excerpt field for a quick summary or your main takeaway. Keep it to 1-2 sentences.
</Callout>

<Callout type="warning">
  The `status` field must be exactly "read", "reading", or "stacked" (lowercase). Other values will cause build errors.
</Callout>
