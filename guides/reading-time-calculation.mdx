The portfolio includes a reading time utility optimized for **Japanese content**. Unlike English text where words are counted, this implementation counts characters, which is more accurate for Japanese.

## Algorithm Overview

The reading time calculator (`src/utils/reading-time.ts:7-24`) performs the following steps:

1. Remove HTML tags
2. Remove code blocks (fenced and inline)
3. Remove whitespace
4. Count remaining characters
5. Calculate minutes based on reading speed

## Implementation

### Core Function

```typescript
export const getReadingTime = (
  content: string,
  wordsPerMinute = 400,
): number => {
  // For Japanese text, count characters instead of words
  // Remove HTML tags, code blocks, and whitespace
  const cleanContent = content
    .replace(/<[^>]*>/g, "")           // Remove HTML tags
    .replace(/```[\s\S]*?```/g, "")   // Remove code blocks
    .replace(/`[^`]*`/g, "")           // Remove inline code
    .replace(/\s+/g, "");              // Remove whitespace

  // Calculate based on character count for Japanese
  const characters = cleanContent.length;
  const minutes = Math.ceil(characters / wordsPerMinute);

  return Math.max(1, minutes);
};
```

### Formatting Function

The utility also includes a formatter for displaying reading time in Japanese:

```typescript
export const formatReadingTime = (minutes: number): string => {
  return `約${minutes}分で読了`;
};
```

This produces output like:
- `約1分で読了` (About 1 minute to read)
- `約5分で読了` (About 5 minutes to read)

## Reading Speed

The default reading speed is **400 characters per minute**, which is a reasonable average for Japanese text. This can be customized:

```typescript
// Slower reading speed (200 chars/min)
const readingTime = getReadingTime(content, 200);

// Faster reading speed (600 chars/min)
const readingTime = getReadingTime(content, 600);
```

## Content Cleaning

### HTML Tag Removal

HTML tags are stripped using a regular expression:

```typescript
.replace(/<[^>]*>/g, "")
```

This ensures that markup doesn't inflate the character count.

### Code Block Removal

Code blocks are removed because:
1. They're read differently than prose
2. They would inflate reading time estimates

```typescript
// Fenced code blocks
.replace(/```[\s\S]*?```/g, "")

// Inline code
.replace(/`[^`]*`/g, "")
```

### Whitespace Removal

For Japanese text, whitespace is typically not significant and is removed:

```typescript
.replace(/\s+/g, "")
```

## Minimum Reading Time

The function always returns at least 1 minute, even for very short content:

```typescript
return Math.max(1, minutes);
```

This prevents displaying "0 minutes" for short posts.

## Japanese vs English

This implementation differs from typical English reading time calculators:

| Aspect | English | Japanese |
|--------|---------|----------|
| **Unit** | Words | Characters |
| **Speed** | 200-250 words/min | 400 chars/min |
| **Whitespace** | Significant | Not significant |
| **Calculation** | Split by spaces | Count all chars |

## Adapting for Mixed Content

If your content mixes Japanese and English, you might want to adjust the algorithm:

```typescript
export const getReadingTime = (content: string): number => {
  const cleanContent = content
    .replace(/<[^>]*>/g, "")
    .replace(/```[\s\S]*?```/g, "")
    .replace(/`[^`]*`/g, "");

  // Count Japanese characters
  const japaneseChars = (cleanContent.match(/[\u3040-\u309F\u30A0-\u30FF\u4E00-\u9FAF]/g) || []).length;
  
  // Count English words
  const englishWords = cleanContent
    .replace(/[^\x00-\x7F]/g, "") // Remove non-ASCII
    .split(/\s+/)
    .filter(w => w.length > 0).length;

  // Combine: 400 chars/min for Japanese, 200 words/min for English
  const minutes = Math.ceil((japaneseChars / 400) + (englishWords / 200));
  
  return Math.max(1, minutes);
};
```
