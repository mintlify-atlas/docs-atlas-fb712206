## Overview

The portfolio includes two RSS features:

1. **RSS Feed Generation** - Automatically generate an RSS feed for your blog posts
2. **RSS Loaders** - Aggregate content from external RSS feeds (Zenn, Note, Podcast)

Both features are built on Astro's content collections and RSS utilities.

## Blog RSS Feed

### Feed Location

Your blog RSS feed is available at:

```
https://yourdomain.com/rss.xml
```

This is generated by `src/pages/rss.xml.ts` and includes all published blog posts.

### Feed Implementation

Here's how the RSS feed is generated (`src/pages/rss.xml.ts:1-24`):

```typescript
import { getCollection } from "astro:content";
import rss from "@astrojs/rss";
import type { APIContext } from "astro";

export async function GET(context: APIContext) {
  const blog = await getCollection("blog");
  const sortedPosts = blog.sort(
    (a, b) => b.data.date.getTime() - a.data.date.getTime(),
  );

  return rss({
    title: "ta93abe | Blog",
    description: "技術ブログ。日々の学びや開発の記録を共有しています。",
    site: context.site?.toString() ?? "http://localhost:4321",
    items: sortedPosts.map((post) => ({
      title: post.data.title,
      pubDate: post.data.date,
      description: post.data.excerpt,
      link: `/blog/${post.id}/`,
    })),
    customData: "<language>ja</language>",
  });
}
```

### Feed Contents

Each feed item includes:

- **title** - The post title from frontmatter
- **pubDate** - The publication date
- **description** - The post excerpt
- **link** - URL to the full post

Posts are sorted by date (newest first).

#### Include Full Content

Instead of just the excerpt, include the full post HTML:

```typescript
import { getCollection } from "astro:content";

export async function GET(context: APIContext) {
  const blog = await getCollection("blog");
  const sortedPosts = blog.sort(
    (a, b) => b.data.date.getTime() - a.data.date.getTime(),
  );

  return rss({
    title: "ta93abe | Blog",
    description: "技術ブログ。日々の学びや開発の記録を共有しています。",
    site: context.site?.toString() ?? "http://localhost:4321",
    items: await Promise.all(
      sortedPosts.map(async (post) => {
        const { Content } = await post.render();
        return {
          title: post.data.title,
          pubDate: post.data.date,
          description: post.data.excerpt,
          content: await Content(),  // Full HTML content
          link: `/blog/${post.id}/`,
        };
      })
    ),
    customData: "<language>ja</language>",
  });
}
```

## RSS Loaders (External Content)

### Overview

RSS loaders fetch and aggregate content from external RSS feeds. This allows you to display articles from platforms like Zenn, Note, or podcasts alongside your own content.

### How It Works

The RSS loader is implemented in `src/utils/rss-loader.ts:1-61`:

```typescript
import Parser from "rss-parser";

export type RSSSource = "Zenn" | "Note" | "Podcast";

interface RSSItem {
  id: string;
  title: string;
  url: string;
  publishedAt: Date;
  excerpt: string;
  source: RSSSource;
}

/**
 * Create an RSS feed loader for Astro content collections
 * @param url - The RSS feed URL
 * @param source - The source name (Zenn, Note, Podcast)
 * @param idPrefix - The prefix for generated IDs (e.g., "zenn", "note", "podcast")
 */
export const createRSSLoader = (
  url: string,
  source: RSSSource,
  idPrefix: string,
) => {
  return async (): Promise<RSSItem[]> => {
    try {
      const parser = new Parser();
      const feed = await parser.parseURL(url);

      if (!feed.items) {
        return [];
      }

      return feed.items
        .filter(
          (
            item,
          ): item is typeof item & {
            title: string;
            link: string;
            pubDate: string;
          } =>
            typeof item.title === "string" &&
            typeof item.link === "string" &&
            typeof item.pubDate === "string",
        )
        .map((item, index) => ({
          id: item.guid ?? `${idPrefix}-${index}`,
          title: item.title,
          url: item.link,
          publishedAt: new Date(item.pubDate),
          excerpt: item.contentSnippet ?? item.content ?? "",
          source,
        }));
    } catch (error) {
      console.error(`Error fetching ${source} RSS feed:`, error);
      return [];
    }
  };
};
```

### External Article Schema

All external content follows the same schema (`src/content.config.ts:76-83`):

```typescript
const externalArticleSchema = (source: RSSSource) =>
  z.object({
    title: z.string(),
    url: z.string().url(),
    publishedAt: z.date(),
    excerpt: z.string(),
    source: z.literal(source),
  });
```

### Adding a New RSS Source

Follow these steps to add a new external RSS feed:

#### 1. Update RSS Source Type

Edit `src/utils/rss-loader.ts` to add your new source:

```typescript
export type RSSSource = "Zenn" | "Note" | "Podcast" | "Medium" | "Dev.to";
```

#### 2. Create Collection

Add a new collection in `src/content.config.ts`:

```typescript
// Medium articles
const medium = defineCollection({
  loader: createRSSLoader(
    "https://medium.com/feed/@username",
    "Medium",
    "medium"
  ),
  schema: externalArticleSchema("Medium"),
});
```

#### 3. Export Collection

Add it to the collections export:

```typescript
export const collections = {
  works,
  books,
  blog,
  talks,
  zenn,
  note,
  podcast,
  medium,  // Add your new collection
};
```

#### 4. Use the Collection

Query the collection in your pages:

```typescript
import { getCollection } from "astro:content";

const mediumArticles = await getCollection("medium");
```


```typescript
import { getCollection } from "astro:content";

const zennArticles = await getCollection("zenn");

const podcasts = await getCollection("podcast");
```

### Combine Multiple Sources

```typescript
const zenn = await getCollection("zenn");
const note = await getCollection("note");

const allExternal = [...zenn, ...note].sort(
  (a, b) => b.data.publishedAt.getTime() - a.data.publishedAt.getTime()
);
```

### Filter by Date

```typescript
const recentArticles = await getCollection("zenn", (article) => {
  const sixMonthsAgo = new Date();
  sixMonthsAgo.setMonth(sixMonthsAgo.getMonth() - 6);
  return article.data.publishedAt > sixMonthsAgo;
});
```

## RSS Feed Best Practices

<Callout type="tip" title="Always Include Excerpts">
  Make sure your blog posts have good excerpts. They appear in RSS readers and help subscribers decide what to read.
</Callout>

<Callout type="info">
  RSS feeds are cached by readers. Changes may take time to appear. Some readers check hourly, others daily.
</Callout>

<Callout type="warning">
  External RSS feeds are fetched at build time. If a feed is down, it will return an empty array but won't break your build.
</Callout>

   ```bash
   npm run dev
   ```

2. **Visit the feed**:
   ```
   http://localhost:4321/rss.xml
   ```

3. **Validate**:

   - Verify dates and links


   - Visit pages that display external content
   - Check that articles load correctly
   - Verify dates and excerpts

## Troubleshooting

### Feed Not Updating

- Clear your browser cache
- Rebuild the site (`npm run build`)
- Check RSS reader cache settings

### External Feed Errors

```bash
Error fetching Zenn RSS feed: ...
```

**Solutions:**

- Verify the RSS URL is correct
- Check if the feed is accessible
- Ensure `rss-parser` is installed: `npm install rss-parser`
- Check for network/firewall issues during build

### Invalid Feed XML

If your generated feed has errors:

- Ensure post titles don't have unescaped HTML
- Check that dates are valid
- Verify URLs are properly formatted
- Use the W3C validator to identify issues
