# Styling

The ta93abe portfolio uses Tailwind CSS 4.x with custom configurations for a consistent, maintainable styling approach.

## Tailwind CSS Setup

### Installation

Tailwind CSS is integrated via the Vite plugin:

**`astro.config.mjs`:**
```javascript
import tailwindcss from "@tailwindcss/vite";
import { defineConfig } from "astro/config";

export default defineConfig({
  vite: {
    plugins: [tailwindcss()],
  },
});
```

### Dependencies

```json
{
  "dependencies": {
    "@tailwindcss/typography": "^0.5.19",
    "@tailwindcss/vite": "^4.1.18",
    "tailwindcss": "^4.1.18",
    "tailwind-merge": "^3.4.1",
    "tailwind-variants": "^3.2.2"
  }
}
```

### Global Styles

Global styles are defined in `src/styles/global.css`.

## CSS Architecture

### Import Structure

**`src/styles/global.css`:**
```css
@import "tailwindcss";
@plugin "@tailwindcss/typography";
@import "@fontsource/shippori-mincho/400.css";
@import "@fontsource/shippori-mincho/500.css";
@import "@fontsource/shippori-mincho/700.css";
```

### File Organization

```
src/styles/
└── global.css        # All global styles
```

Component-specific styles are defined in `<style>` blocks within `.astro` files.

## Theming System

### CSS Custom Properties

The site uses CSS custom properties for theming:

```css
:root {
  /* Primary Color */
  --primary-color: #4f46e5;
  
  /* Backgrounds */
  --bg-primary: #ffffff;
  --bg-secondary: #f8fafc;
  --bg-tertiary: #f1f5f9;
  
  /* Text Colors */
  --text-primary: #1a1a1a;
  --text-secondary: #4b5563;
  --text-muted: #9ca3af;
  
  /* Borders */
  --border-color: #e2e8f0;
  
  /* Links */
  --link-color: #2563eb;
  --link-hover-color: #1d4ed8;
  
  /* Content-Specific */
  --blockquote-bg: #eff6ff;
  --blockquote-border: #3b82f6;
  --code-bg: #f1f5f9;
  --code-text: #1e293b;
}
```

### Usage in Components

```css

  background: var(--bg-primary);
}

.hero-title {
  color: var(--text-primary);
```

The `@tailwindcss/typography` plugin provides prose styling:


## Accessibility

### Reduced Motion

```css
@media (prefers-reduced-motion: reduce) {
  *,
  *::before,
  *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
```

## Animations

### Scroll-Triggered Fade-In

```css
[data-animate] {
  opacity: 0;
  transform: translateY(20px);
  transition:
    opacity 0.6s ease,
    transform 0.6s ease;
}

[data-animate].is-visible {
  opacity: 1;
  transform: translateY(0);
}
```

**Respects Reduced Motion:**
```css
@media (prefers-reduced-motion: reduce) {
  [data-animate] {
    opacity: 1;
    transform: none;
    transition: none;
  }
}
```

## Utility Libraries

### Tailwind Merge

Combine Tailwind classes efficiently:

```typescript
import { twMerge } from "tailwind-merge";

const className = twMerge(
  "px-4 py-2",
  "px-6"  // Overrides px-4
);
```

### Tailwind Variants

Create variant-based component styles:

```typescript
import { tv } from "tailwind-variants";

const button = tv({
  base: "inline-flex items-center justify-center",
  variants: {
    variant: {
      primary: "bg-neutral-900 text-white",
      secondary: "bg-transparent border-2",
    },
  },
});
```

## Scoped Component Styles

Components can include scoped styles:

```astro
<style>
  .hero-section {
    min-height: 100dvh;
  }
  
  @media (max-width: 768px) {
    .hero-section {
      min-height: 100svh;
    }
  }
  
  @media (prefers-reduced-motion: reduce) {
    a, button {
      transition: none;
    }
  }
</style>
```

## Responsive Design

### Breakpoints

Tailwind CSS default breakpoints:

```css
/* sm: 640px */
/* md: 768px */
/* lg: 1024px */
/* xl: 1280px */
/* 2xl: 1536px */
```

### Responsive Typography

Using `clamp()` for fluid typography:

```css
.hero-title {
  font-size: clamp(3rem, 10vw, 7rem);
}

.hero-description {
  font-size: clamp(1.125rem, 2.5vw, 1.5rem);
}
```

### Viewport Units

```css
.hero-section {
  min-height: 100dvh;  /* Dynamic viewport height */
}

@media (max-width: 768px) {
  .hero-section {
    min-height: 100svh;  /* Small viewport height */
  }
}
```

## Dark Mode

Currently, the site uses a light theme only. Dark mode is not implemented but can be added using:

```css
@media (prefers-color-scheme: dark) {
  :root {
    --bg-primary: #1a1a1a;
    --text-primary: #ffffff;
    /* ... other dark theme colors */
  }
}
```

## Best Practices

1. **Use CSS Variables** - For theme values that may change
2. **Tailwind First** - Use Tailwind utilities before custom CSS
3. **Scoped Styles** - Keep component styles in `<style>` blocks
4. **Responsive** - Mobile-first approach
5. **Accessible** - Include focus and reduced motion styles
6. **Semantic** - Use semantic color names
7. **Performance** - Minimize custom CSS
