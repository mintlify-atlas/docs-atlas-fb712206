## Key Design Patterns

### Component Organization

Components are organized by domain:
- **`ui/`** - Reusable design system components
- **`blog/`** - Blog-specific features
- **`landing/`** - Landing page sections
- Root level for layout components (Header, Footer)

### Content-Driven

Content is managed through Astro Content Collections:
- Type-safe content with schemas
- Markdown/MDX support
- Organized by content type

### Utility-First

Utility functions are:
- Pure and testable
- Domain-specific (date, books, content)
- Located in `src/utils/`

### File-Based Routing

Pages follow Astro's file-based routing:
- `pages/index.astro` в†’ `/`
- `pages/blog/[id].astro` в†’ `/blog/:id`
- `pages/rss.xml.ts` в†’ `/rss.xml`
