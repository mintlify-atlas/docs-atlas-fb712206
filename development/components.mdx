# Components

The ta93abe portfolio uses Astro components to build a modular, reusable UI. Components are organized by category and follow consistent patterns.

## Component Architecture

### Component Categories

Components are organized into three main categories:

1. **UI Components** (`src/components/ui/`) - Generic, reusable design system components
2. **Blog Components** (`src/components/blog/`) - Blog-specific features
3. **Landing Components** (`src/components/landing/`) - Landing page sections
4. **Layout Components** (root level) - Site structure components

## UI Components

Generic components that form the design system.

### Button (`ui/Button.astro`)

A versatile button component with variant and size options.

**Props:**
```typescript
interface Props {
  variant?: "primary" | "secondary" | "ghost";
  size?: "sm" | "md" | "lg";
  href?: string;
  disabled?: boolean;
  class?: string;
  type?: "button" | "submit" | "reset";
}
```

**Usage:**
```astro
import Button from "../components/ui/Button.astro";

<Button variant="primary" size="lg" href="/works">
  View Works
</Button>

<Button variant="secondary" size="md">
  Click Me
</Button>

<Button variant="ghost" size="sm" disabled>
  Disabled
</Button>
```

**Features:**
- Three variants: primary, secondary, ghost
- Three sizes: sm, md, lg
- Can render as `<a>` or `<button>` based on `href` prop
- Hover effects with translate and shadow
- Focus-visible outline
- Respects `prefers-reduced-motion`
- Uses `tailwind-variants` for style composition

### Callout (`blog/Callout.astro`)

Highlight boxes for important information.

**Props:**
```typescript
interface Props {
  type?: "info" | "warning" | "danger" | "tip";
  title?: string;
}
```

**Usage:**
```astro
import Callout from "../components/blog/Callout.astro";

<Callout type="info" title="Note">
  This is an informational callout.
</Callout>

<Callout type="warning">
  Warning: Proceed with caution.
</Callout>

<Callout type="danger" title="Important">
  Critical information here.
</Callout>

<Callout type="tip">
  Pro tip: Use this feature!

```

**Features:**
- Four types with color-coded styling
- Custom or default titles
- Icon per type
- Bordered left accent
- Colored backgrounds

## Component Patterns

### Props Interface

All components define TypeScript interfaces for props:

```astro
---
interface Props {
  variant?: "primary" | "secondary";
  size?: "sm" | "md" | "lg";
  class?: string;
}

const { 
  variant = "primary", 
  size = "md",
  class: className = ""
} = Astro.props;
---
```

### Style Composition

Components use `tailwind-variants` for composable styles:

```typescript
import { tv } from "tailwind-variants";

const button = tv({
  base: "inline-flex items-center justify-center rounded-full",
  variants: {
    variant: {
      primary: "bg-neutral-900 text-white",
      secondary: "bg-transparent border-2",
    },
    size: {
      sm: "px-4 py-2 text-sm",
      md: "px-6 py-2.5 text-base",
    },
  },
  defaultVariants: {
    variant: "primary",
    size: "md",
  },
});
```

### Slot Usage

Components use Astro's slot system:

```astro
<div class="card">
  <slot />
</div>
```

### Conditional Rendering

Components handle conditional rendering in the template:

```astro
{
  href && !disabled ? (
    <a href={href} class={classes}>
      <slot />
    </a>
  ) : (
    <button type={type} disabled={disabled} class={classes}>
      <slot />
    </button>
  )
}
```

### Scoped Styles

Components use scoped `<style>` blocks:

```astro
<style>
  .hero-section {
    min-height: 100dvh;
  }
  
  @media (max-width: 768px) {
    .hero-section {
      min-height: 100svh;
    }
  }
</style>
```

### Accessibility

Components include accessibility features:
- Focus-visible outlines
- ARIA attributes
- Semantic HTML
- Keyboard navigation support
- Reduced motion support

## Component Composition

Components are composed to build complex UIs:

```astro
---
import Layout from "../layouts/Layout.astro";
import HeroSection from "../components/landing/HeroSection.astro";
import Card from "../components/ui/Card.astro";
import Button from "../components/ui/Button.astro";
---

<Layout title="Home">
  <HeroSection />
  
  <section>
    <Card>
      <h2>Featured Work</h2>
      <p>Check out my latest projects</p>
      <Button href="/works">View All</Button>
    </Card>
  </section>
</Layout>
```

## Best Practices

1. **Single Responsibility** - Each component has one clear purpose
2. **Props Over Slots** - Use props for data, slots for content
3. **TypeScript Interfaces** - Always define prop types
4. **Default Values** - Provide sensible defaults
5. **Composability** - Build complex UIs from simple components
6. **Accessibility** - Include ARIA and semantic HTML
7. **Responsive Design** - Use responsive utilities and media queries
8. **Performance** - Keep components lightweight
